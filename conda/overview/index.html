    Conda

    Conda is an open-source package and environment management tool that is available to use on Alma. It is language agnostic, meaning that it is a tool that can be used to manage a variety of languages; including Python, R and Perl, for example.

    There are multiple implementations of conda with slight differences. The base implementation of the software is known as Anaconda, though there is a highly efficient alternative known as Mamba which can offer orders of magnitude speed up when resolving complex environments.

    Alma users are encouraged to use Mamba, rather than Anaconda when they create environments and install packages for this reason.

    How to setup conda

    1. Load the Mamba module

    In order to use conda, users should add the EasyBuild modules to their user path and load the Mamba module:

    module use /opt/software/easybuild/modules/all
module load Mamba

    You can check that the module has been loaded successfully by running module list, which should return the output:

    Currently Loaded Modules:
  1) slurm-21.08.5   2) Mamba/23.1.0-0

    in addition to any other modules you may have already loaded. You can further check that the mamba and conda commands are now available for you to use, via mamba --version, which should return the output:

    mamba 1.4.0
conda 23.1.0

    2. Initialize your shell

    After the module has been loaded, you should now initialize your shell. The easiest way to achieve this, is to run

    mamba init

    3. Using conda post-initialization

    After you have started a fresh terminal session, you do not need to follow the previous steps again. Check that you can run

    mamba --version

    which as before, should now return:

    mamba 1.4.0
conda 23.1.0

    You should now be ready to use conda.
